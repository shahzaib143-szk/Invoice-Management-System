## Why the files feel so discrete

Because each file has a narrow responsibility.

### FXML files

Only define layout and controls.

### Controllers

Only handle user actions and connect UI to services.

### Services

Only apply rules, calculations, validation, and workflows.

### DAOs

Only read/write the database.

### Models

Only hold data.

That separation is why the code feels split into many pieces. It is not a problem. It is the architecture.

## What must be true for these screens to work

These FXML files are real GUI files, but they are only fully working if all of this is also true:

* `fx:controller` points to the correct controller class
* every `fx:id` exists as a matching `@FXML` field in the controller
* every `onAction="#..."` method exists in the controller
* `MainApp` or `SceneNavigator` actually loads these FXML files with `FXMLLoader`
* the controllers fill the tables and combo boxes correctly at runtime

So the screens exist. The next question is wiring.
